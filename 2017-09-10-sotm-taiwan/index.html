把政府開放資料整合進 OSM

問題一

資料對比

  • 要人工對比資料
  • 朝陽科技大學的案例

問題一

資料對比

    • 耗時間，而且不容易掌控進度。
    • 需要少數人特別花心力。
    • 做過一次大概不會想再做第二次（第三次、第四次⋯⋯）

問題二

資料合併

  • 如果 OSM 上面已經有這筆資料，把兩筆資料合併起來要多花一點工夫。
  • 做過一次大概不會想再做第二次（第三次、第四次⋯⋯）
  • 所以不容易與政府資料保持同步。

我的提案

我的提案是接下來要說明的這個工具。

不過還在開發中，目前只有測試版。

tasks.lowiki.org

給 OSM Tasking Manager 加一個 Map data 的功能。

清楚的工作流程

  • 小塊的工作
  • 可見的進程
  • 清楚的目標
  • 規模化

這些都是 Tasking Manager 的優點
（感謝 HOT Team）

資料更新的時候

  • 如果工作項目的資料沒有變，它的狀態也維持不變。
  • 如果工作項目的資料更新了，它的狀態會重置。

  • 只需要挑出資料有更新的工作項目來處理。

自動化整合

  • 我們可以開始把政府開放資料轉檔，定期導入 Tasking Managerv
  • Mapper 只要偶爾來看一下有沒有新資料進來，需不需要編輯。
  • Import 可以變成隨時持續進行的工作。
