一、建置 LocalWiki
維基地方誌系統

維基地方誌網站已建置於 https://lowiki.tw/

 
 

網站內容架構

  1. 以「村里」行政層級為地區。
  2. 各地區包含多個頁面。
  3. 各頁面包含該地點相關資料與地圖。
  4. 以標籤進行分類。
  5. 提供全文檢索功能。
  6. 預先建置全台灣所有村里與災防相關設施資料。
 
 
 
 
 

二、整合開放資料與開放地理圖資

整合開放資料，應俱備以下功能：

  • 導入 OpenStreetMap 地理圖資
  • 引導防災地方誌使用者編修 OpenStreetMap

導入 OpenStreetMap 地理圖資

查詢 OpenStreetMap 所得的地理圖資，可直接選用以導入 LocalWiki 資料庫中。

 
 

引導防災地方誌使用者編修 OpenStreetMap

如果查詢 OpenStreetMap 無可用之結果，網站會提示使用者，並提供連結至 OpenStreetMap 編修其資料。

 
 

三、整合地圖網路服務

防災地方誌網站應提供網路地圖，並應俱備以下功能：

  • 地圖可以捲動、縮放、定位。
  • 可選擇 WMS、WFS 等協定之底圖圖層。
  • 可在地圖上添加、編修、刪除幾何物件。
  • 可以用地名或地址搜尋 OpenStreetMap 相關圖資匯入編輯器。
 
 
 
 
 

四、網頁模板功能

網站應提供以下模板功能：

  • 每個地區都有一組預設的模板可以選用。
  • 模板中將提供各類型標的物的屬性表格，供一般使用者編輯時填寫，簡化建置文件的流程。
  • 模板中能填寫的內容，以文字為限。
  • 一般使用者可以自行編修模板，也可以增加新的模板。
  • 地區管理者可以鎖定特定模板，不提供一般使用者自行編修。
  • 一般使用者編輯文件時，可以在編輯器中插入該地區的模板並填寫變數資料。
  • LocalWiki 系統在顯示頁面時，將自動代入模板進行編排。
  • LocalWiki 標籤可設定相關的模板。一般使用者編輯或新建文件時，可選擇要加入的標籤分類。系統會自動將該標籤相關的模板，插入文件開頭，供編輯運用。

十、壓力測試

  • 使用 Locust 網站效能測試工具，透過 HTTP 與 HTTPS 通訊協定。
  • 設定為 100 位使用者同時發送請求。
  • 共測試 5 組 URL 網址，包含平台功能：開啟區域首頁、查詢地圖、查詢標籤、編輯頁面、編輯區域首頁。
  • 持續測試到對每一組 URL 網址發送超過 100 個請求。
